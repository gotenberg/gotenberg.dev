[Gotenberg](https://github.com/gotenberg/gotenberg) is a Docker-powered API for converting documents to PDF.

It acts as a self-contained wrapper around powerful tools like **Chromium** and **LibreOffice**, exposing them through a simple API. This eliminates the need to install and manage heavy dependencies in your application environment.

## How it Works

Instead of managing Chromium or LibreOffice binaries in your own backend, you send your files (HTML, Markdown, Word, Excel, etc.) to the Gotenberg container via `multipart/form-data`. Gotenberg handles the rendering and returns a PDF.

## Why Gotenberg?

- **Chromium at the Core:** It uses the Chromium engine (headless) to render HTML/CSS/JS exactly as it appears in a browser.
- **Office Support:** Converts `.docx`, `.xlsx`, and `.pptx` using the LibreOffice engine.
- **PDF Manipulation:** Merge, split, encrypt PDFs using robust engines like pdfcpu and QPDF.
- **Zero Dependency Hell:** It runs as a single Docker image. You don't need to worry about fonts, shared libraries, or browser versions.

## Quick Start

You can get a PDF from a URL with a single command:

```bash
# 1. Start the container.
docker run --rm -p "3000:3000" gotenberg/gotenberg:8

# 2. Convert a URL to PDF.
curl \
--request POST http://localhost:3000/forms/chromium/convert/url \
--form url=https://sparksuite.github.io/simple-html-invoice-template/ \
-o invoice.pdf
```
